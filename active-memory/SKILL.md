# AI 主动记忆操作技能

## 技能用途

此技能提供 AI 主动记忆读写的**完整操作规范**。Rule 层（`active-memory-recording.mdc`）已判断当前场景需要执行记忆操作，并路由到此技能。

## 技能基础路径解析

本技能的文件引用均使用相对路径。AI 执行时必须先确定技能的实际安装路径（`SKILL_BASE`）：

**路径发现顺序**（按优先级）：
1. 当前已加载的 SKILL.md 所在目录（首选，直接可用）
2. `~/.codebuddy/skills/active-memory/`

确定 `SKILL_BASE` 后，所有 `read_file` 调用使用 `${SKILL_BASE}/<相对路径>` 格式。

## 记录什么【必须】

AI 必须主动识别并记录以下关键信息到 `.codebuddy/memory/` 目录：

1. **关键结论**：经分析、讨论、验证后得出的重要结论
2. **解决方案**：已确认的技术方案、架构决策、实现策略
3. **决策点**：用户的关键选择、确认/否决的方案及理由
4. **核心认知**：对项目架构、业务逻辑、技术栈的深入理解
5. **待办事项**：待实施的任务、待确认的疑问

**禁止记录**：临时搜索结果、工具错误、重复内容、敏感信息（除非明确要求）

## 入口判断与路由

技能启动时，根据触发场景决定执行路径：

### 读取路径

触发条件：新会话开始、询问历史上下文、继续未完成任务、引用已有决策

**【必须】执行 `read_file` 读取 `${SKILL_BASE}/flows/read-flow.md`，按该文件指引执行读取操作。**

### 写入路径

触发条件：方案确认、重要结论达成、架构认知突破、解决技术问题、关键决策记录、阶段性成果、方案否决

**执行顺序【必须】：**
1. **先分类**：执行 `read_file` 读取 `${SKILL_BASE}/flows/classify-flow.md`，判断信息应归入长期记忆还是近期日志
2. **再写入**：执行 `read_file` 读取 `${SKILL_BASE}/flows/write-flow.md`，按分类结果执行对应的写入操作

### 需要参考示例时

首次写入或不确定格式时，额外读取 `${SKILL_BASE}/templates/examples.md` 获取记录示例和质量自检清单。

## 执行原则【必须】

1. **主动性**：不等用户要求，主动识别和记录
2. **及时性**：关键信息产生时立即记录
3. **简洁性**：提炼核心信息，避免冗余
4. **不干扰**：记录过程不打断当前任务流程
5. **协同性**：与 `<working_memory_files>` 系统规则协同（本技能为其细化补充）

## 文件结构

```
active-memory/
├── SKILL.md                  ← 入口路由（本文件）
├── flows/
│   ├── read-flow.md          ← 读取流程详细规范
│   ├── classify-flow.md      ← 分类决策流程（长期/近期划分标准）
│   └── write-flow.md         ← 写入流程 + 格式模板 + 错误处理
└── templates/
    └── examples.md           ← 记录示例 + 质量自检清单
```
