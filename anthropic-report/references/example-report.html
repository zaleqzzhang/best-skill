01

Hermes Agent 闭环学习系统

Nous Research 的 Hermes Agent 是目前唯一内置学习闭环的开源 AI Agent。它不只是执行任务，而是从每次任务中提取可复用知识，形成"程序化记忆"。

Hermes Agent 的技能系统不是简单的"把步骤存下来"。它构建了一个有门槛筛选、有条件激活、有增量改进、有容量约束的完整闭环。

几个精妙的设计点：技能只在满足条件时才创建（≥5 次工具调用、解决了 error、被用户纠正），避免垃圾技能泛滥；技能分三级加载（L0 目录摘要仅 ~3k tokens），解决了"技能多了 context 爆炸"的问题；MEMORY.md 限制 2200 字符的有界记忆设计，迫使 agent 不断做知识蒸馏。

核心机制
与传统"离线训练 + 在线推理"范式不同，Hermes 在实际 task-solving 过程中，通过每一个工具调用的结果反馈来调整知识表达。这是一种运行时的、增量式的、有界的学习。
02

Hermes vs OpenClaw：深度优先 vs 广度优先

两者走了完全不同的路线——OpenClaw 追求生态广度和平台覆盖，Hermes 追求学习深度和自我进化。它们不是替代关系，而是互补关系。

"好不好用"取决于你要什么。如果追求 agent 越用越聪明，Hermes 确实做得更深——它的技能是从实战中"长出来的活文档"，而 OpenClaw 的技能是"从市场买回来的静态资源"（其中 341 个还是恶意的）。

但 OpenClaw 的生态优势也是真实的：50+ 消息平台接入、345K Stars 的社区、2857+ 预制技能。两者更像互补关系而非替代关系。

03

AI Coding 的三层借鉴

Hermes 的技能系统不是为 coding 设计的，但它的机制对 AI Coding 工作流有三层递进的启示——从静态规则到活知识，从重复推理到模式复用，从失忆到经验积累。

当前 AI Coding 工具的一个核心痛点是：每次对话都从零开始推理。上周花 30 分钟排查的 ESM/CJS 兼容问题，这周遇到类似错误，agent 又完全不记得。

三层借鉴是递进的——第一层解决"规范记忆"，第二层解决"模式复用"，第三层解决"经验积累"。当 coding agent 能做到第三层时，它就不再是一个工具，而是一个真正的同事。

核心启示
停止让每次对话都从零开始。让 coding agent 积累项目级的程序化记忆——不是静态规则文件，而是从实战中自动生长、持续改进的活知识。
04

OpenClaw 企业级部署的三代演进

从"每人一台 VPS"到 Kubernetes 原生编排，企业级 AI Agent 部署正在经历快速进化——但根本的安全模型问题仍未解决。

"每人一台 VPS"的时代正在过去。ClawManager 作为首个开源的 K8s 原生方案，将百人团队的环境搭建时间从半天压缩到 5 分钟，并提供了从资源隔离到全链路审计的完整能力。

但技术方案只是表层。微软安全团队 2 月的报告指出了更根本的问题：OpenClaw 能执行任意代码、能安装外部技能、能持久化凭据——无论怎么套壳隔离，这些特性在企业环境里就是定时炸弹。341 个恶意技能已经不是理论风险。

微软的建议
将 OpenClaw 视为可丢弃的隔离环境运行，使用专用凭据，监控记忆篡改，随时准备销毁重建。很多企业最终选择"用 OpenClaw 的理念，但不用 OpenClaw 本身"。
  • K8s 自建：ClawManager（MIT），适合有运维能力的团队
  • 容器化 VPS：Docker Compose + 反向代理，适合快速起步
  • 托管平台：Tensol / Viktor，适合不想自运维的企业
  • 云厂商方案：腾讯云 ADP / 阿里云，适合已有云资源的企业

写在最后

Hermes Agent 的真正创新不是"把步骤存成文件"，而是围绕技能构建了一个有门槛筛选、有条件激活、有增量改进、有容量约束的完整闭环。

Core Takeaway
停止让每次对话都从零开始。让 coding agent 积累项目级的程序化记忆——不是静态规则文件，而是从实战中自动生长、持续改进的活知识。当 agent 学会了"记得上次的坑"，它就完成了从工具到同事的质变。

当然也要清醒地看到局限：当前机制主要作用于知识层面（procedural memory），而非 reasoning 层面。agent 可以学习"怎么做"，但很难自主学习"怎么想"。

真正的圣杯是后者——让 agent 学会这个项目的"思考方式"，而不只是"操作步骤"。这条路还很长，但 Hermes 指出了一个值得探索的方向。
