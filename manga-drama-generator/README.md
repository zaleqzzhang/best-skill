# AI 漫剧全流程生成技能（manga-drama-generator）

## 功能说明

本技能将小说内容**一站式转化**为 AI 动漫短剧制作所需的完整素材包，覆盖从文字到视觉的全链路。整个流程包括 5 个阶段：

1. **小说转剧本** — 以剧本改编专家身份，将小说文本解析为标准影视剧本格式
2. **剧本处理** — 自查 → 大纲 → 分场大纲 → 分场剧本，确保节奏、钩子、卡点到位
3. **人物信息处理** — 提取人物信息、撰写小传、提炼视觉关键词、生成三视图提示词
4. **场景信息处理** — 提取场景列表、生成场景绘图提示词（与阶段 3 可并行）
5. **分镜脚本生成** — 按 8 条核心原则拆分镜头，生成首帧/尾帧/视频提示词
6. **输出 Excel 表格** — 汇总为人物信息表、场景信息表、分镜信息表三张工作表

适用场景：小说转漫剧、动漫短剧制作、AI 绘图提示词生成、分镜脚本设计。

---

## 详细输入输出

### 输入

| 参数 | 必填 | 类型 | 说明 |
|------|:----:|------|------|
| 小说内容 | ✅ | 文本/文件 | 粘贴文本或上传小说文件 |
| 动漫风格 | ✅ | 字符串 | 从可选风格列表中选择（见下方） |
| 改编程度 | ❌ | 字符串 | `忠实原著` / `适度改编` / `创意改编`，默认"适度" |

**可选动漫风格**：

> 美式卡通、2D古风、3D古风、韩漫二次元、现代都市、3D卡通、日漫二次元、中国工笔画、写实风格、彩色水墨、厚涂古风、吉卜力、赛博朋克、胶片质感

若未指定风格，系统会根据小说题材推荐合适风格并征得确认。

### 输出

最终输出一个 Excel 文件 `{小说文件名}_完整表格.xlsx`，包含三张工作表：

#### 表格 1：人物信息表

| 列名 | 说明 |
|------|------|
| 人物编号 | 按戏份从多到少排序 |
| 人物姓名 | 角色名称 |
| 人物信息提取 | 姓名、性别、年龄段、社会身份、性格标签、关系、反差点、记忆点 |
| 人物小传 | 150-200 字的角色小传 |
| 人物三视图提示词 | 21:9 宽幅构图的中文绘图提示词（含面部特写 + 正/侧/后视图） |

#### 表格 2：场景信息表

| 列名 | 说明 |
|------|------|
| 场景编号 | S001 起始，按出现顺序排列 |
| 场景名字 | 简洁的场景命名 |
| 场景提示词 | 21:9 宽幅空镜头的中文绘图提示词 |

#### 表格 3：分镜信息表

| 列名 | 说明 |
|------|------|
| 第几集 | 集数编号 |
| 第几场 | 场次编号 |
| 第几个镜头 | 镜头序号 |
| 时长(s) | 1-10 秒 |
| 景别 | 远景/全景/中景/近景/特写/大特写 |
| 摄法 | 固定/横摇/俯仰摇/推/拉/变焦/手持/跟随 |
| 画面内容 | ≥50 字，融合视觉风格、环境、人物、动作、情绪、光影、构图、特效 8 大要素 |
| 台词/音效 | 对白或音效标注 |
| 入镜角色 | 多人用中文逗号隔开 |
| 场景标识 | 场景名称 |
| 首帧提示词 | 首帧图片的中文绘图提示词 |
| 尾帧提示词 | 尾帧图片的中文绘图提示词 |
| 视频提示词 | 图生视频的中文提示词 |

此外，流程中间阶段还会产出：
- 标准格式剧本（场景 + 对白 + 动作描述）
- 剧本大纲（含叙事结构选择）
- 分场剧本（视听化的最终剧本）

## 使用说明

### 前置依赖

```bash
pip install pandas openpyxl
```

（脚本缺少依赖时会自动安装）

### 全流程执行

整个流程为交互式执行，每个关键阶段完成后会暂停等待用户确认：

```
小说输入 → 阶段1(剧本) → 确认 → 阶段2(剧本处理) → 确认
         → 阶段3A(人物) + 阶段3B(场景) 并行 → 确认
         → 阶段4(分镜) → 阶段5(Excel输出) → 完成
```

**用户确认点**：
1. 阶段 1 完成后：展示初版剧本摘要（场景数、角色数、总时长估算）
2. 阶段 2.2 完成后：展示剧本大纲，确认叙事结构
3. 阶段 2.4 完成后：展示分场剧本
4. 阶段 3 完成后：展示人物设定和场景列表摘要
5. 阶段 5 完成后：确认 Excel 文件已生成

### Excel 生成命令

一键生成全部三张表：

```bash
python3 scripts/generate_all_tables.py \
  --characters characters.json \
  --scenes scenes.json \
  --storyboards storyboards.json \
  --project-name "<小说名>"
```

也可单独生成：

```bash
python3 scripts/generate_character_table.py   # 人物信息表
python3 scripts/generate_scene_table.py       # 场景信息表
python3 scripts/generate_storyboard_table.py  # 分镜信息表
```

## 使用示例

### 示例 1：完整流程

```
用户：请帮我把这段小说转成漫剧素材，风格选"赛博朋克"

[粘贴小说文本...]

AI：收到！我将以"赛博朋克"风格、"适度改编"程度开始处理。

--- 阶段1：小说转剧本 ---
已生成剧本，共 5 个场景、8 个角色，预估总时长约 6 分钟。
[展示剧本摘要...]
请确认是否满意，或提出修改意见。

用户：可以，继续。

--- 阶段2：剧本处理 ---
选用"费希特曲线"叙事结构，已生成 4 集分场剧本...
[展示大纲和分场剧本...]

用户：第 2 集的钩子不够强，加点悬念。

AI：已修改第 2 集开头，增加了悬念元素...

--- 阶段3A+3B：人物 & 场景处理（并行）---
提取到 8 个角色、12 个场景...
[展示人物设定和场景列表...]

--- 阶段4：分镜脚本生成 ---
共生成 48 个镜头，含首帧/尾帧/视频提示词...

--- 阶段5：输出 Excel ---
已生成文件：赛博朋克小说_完整表格.xlsx
包含：人物信息表(8条) | 场景信息表(12条) | 分镜信息表(48条)
```
