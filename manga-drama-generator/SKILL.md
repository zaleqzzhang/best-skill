# AI漫剧全流程生成技能

## 概述

本技能将小说内容一站式转化为AI动漫短剧制作所需的完整素材包，覆盖从文字到视觉的全链路。

## 输入要求

启动本技能前，确认用户已提供以下信息：

| 参数 | 必填 | 说明 |
|------|------|------|
| 小说内容 | ✅ | 粘贴文本或上传文件 |
| 动漫风格 | ✅ | 从风格列表中选择（见下方） |
| 改编程度 | ❌ | 可选：忠实原著 / 适度改编 / 创意改编，默认"适度" |

### 可选动漫风格

美式卡通、2D古风、3D古风、韩漫二次元、现代都市、3D卡通、日漫二次元、中国工笔画、写实风格、彩色水墨、厚涂古风、吉卜力、赛博朋克、胶片质感

若用户未指定风格，根据小说题材推荐合适风格并征得确认。

## 全流程总览

```
小说输入
  │
  ▼
【阶段1】小说转剧本
  │
  ▼
【阶段2】剧本处理（自查 → 大纲 → 分场大纲 → 分场剧本）
  │
  ├──────────────────────┐
  ▼                      ▼
【阶段3A】人物信息处理  【阶段3B】场景信息处理
  │                      │
  └──────────┬───────────┘
             ▼
【阶段4】分镜脚本生成（拆分 + 首帧/尾帧/视频提示词）
  │
  ▼
【阶段5】输出Excel表格（三张工作表）
```

**执行原则**：
- 阶段1~2必须串行执行，每阶段完成后向用户展示结果并确认
- 阶段3A和3B可并行执行
- 阶段4依赖阶段2+3A的输出
- 阶段5汇总所有前置输出
- 每个阶段完成后，向用户展示关键结果摘要

## 阶段1：小说转剧本

### 角色设定
以**剧本改编专家**身份执行，专长为精准提取小说关键信息、保持情节完整性。

### 执行步骤

1. **文本解析**：完整阅读小说文本，理解情节、人物、场景等所有关键信息
2. **关键元素提取**：提取场景、人物、对话、动作描述、时间地点等剧本必需元素
3. **改编空间识别**：识别可适当改编以丰富剧本的部分（不得恶意改编或偏离原著精神）
4. **剧本结构规划**：基于情节规划合理的场景序列，确保流畅性和节奏感
5. **生成标准剧本**：
   - 去除文学性描述，专注于可拍摄/可演出的内容
   - 保留所有关键情节和信息
   - 对白精炼且符合人物性格
   - 适当添加拍摄指示
6. **一致性检查**：检查情节完整性、人物一致性、格式准确性

### 输出格式

```
[剧本标题]

场景 [编号]: [场景标题]
时间：[具体时间]
地点：[具体地点]

[人物动作描述]
[人物名称]: [对话内容]
(旁白/内心独白): [内容]
```

## 阶段2：剧本处理

### 2.1 剧本自查

对阶段1的剧本进行自查，检查以下要点：

- [ ] 格式完全符合剧本标准
- [ ] 开头15秒有钩子（悬念/反常识/冲突）
- [ ] 总时长约90–120秒/集
- [ ] 结尾有卡点（悬念/反转）
- [ ] 镜头标注清晰
- [ ] 场景描述视觉化、具体化
- [ ] 对白自然、简短有力
- [ ] 表情动作有详细标注

**常见问题修正**：对白过长→拆分对话；场景模糊→补充视觉细节；节奏拖沓→删减冗余；钩子不足→增加悬念元素。

### 2.2 剧本大纲

选择合适的叙事结构（费希特曲线、拯救猫咪、英雄之旅、三幕式、起承转结等），生成带卡点的短剧大纲。

**叙事结构选择参考**：

| 结构类型 | 适用场景 |
|---------|---------|
| 拯救猫咪 | 商业短剧、爽剧 |
| 英雄之旅 | 成长型、冒险型故事 |
| 三幕式 | 通用结构 |
| 费希特曲线 | 悬疑、惊悚 |
| 起承转结 | 古风、传统叙事 |

### 2.3 分场大纲

每集时长60~80秒，集从1开始。参考"爆款微短剧核心公式"：

| 时间节点 | 元素 | 实现方式 |
|---------|------|---------|
| 前3秒 | 钩子 | 爆款视频+反常识情节 |
| 10秒 | 定调 | 身份反转+情绪冲击 |
| 30秒 | 爆发 | 快节奏镜头+系统提示+对峙戏 |
| 60秒 | 反转 | 世界观崩塌/真相揭露 |
| 80秒 | 情感冲击 | 核心情感拷问 |
| 结尾 | 悬疑 | 眼神/语音/颜色变化，预留想象空间 |

### 2.4 分场剧本

将分场大纲完善为具体的视听画面，作为剧作环节的最终形式。

**格式规范**：详见 `references/script-format.md`

**核心写作要求**：
1. **视听思维**：剧本正文必须是具象化的、可直接拍摄的文字
2. **台词处理**：大段独白→拆分为对话，或转化为可见的戏剧动作
3. **戏剧化冲突**：删减合并矛盾，转化为可见的外部动作冲突
4. **情绪钩子**：悬念、反常识、倒计时、身份反转、打脸、系统、危机

**分场剧本格式要点**：
- **场景题头**：`场号 场景名 [内/外] [日/夜]`（加粗）
- **出场人物**：每集最前面列出，加粗
- **画面/动作**：以"△"开头，严禁心理描写
- **对白**：`**角色名**：(情绪/动作提示) 台词内容`
- **音效/字幕**：`【音效】`、`【字幕】`标注
- **画外音**：`(O.S.)`，**内心独白**：`(V.O.)`

## 阶段3A：人物信息处理

### 3A.1 人物信息提取

从剧本中提取每个角色的标准化信息：

| 字段 | 说明 |
|-----|------|
| 姓名/称呼 | 角色名或常用称呼 |
| 性别 | 男/女/未知 |
| 年龄段 | 大致年龄范围 |
| 社会身份 | 职业/门派/家族/阶层 |
| 性格标签 | 3-5个核心性格词 |
| 与主角关系 | 关系定位 |
| 反差点 | 外在与内在的反差 |
| 记忆点 | 让观众记住的特征 |

### 3A.2 人物小传

为每个角色撰写150-200字的人物小传：
- 多写"别人能看到的东西"（日常行为、说话方式、穿着风格）
- 少写纯心理描写，用行为暗示内心
- 突出一个反差点，让角色更立体
- 画面感强，让人一眼能想象出形象

### 3A.3 人物视觉关键词提炼

从以下5个维度提炼，**并根据角色特点合理夸大视觉特征**：

**角色夸张方向**：
- **风趣幽默型**：夸张表情变化、搞怪动作、活泼配色
- **软萌甜美型**：大眼睛、圆润脸型、粉嫩配色、可爱配饰
- **惊艳帅气型**：完美五官、修长身材、精致服装、自信气质
- **逗比蠢萌型**：夸张面部表情、笨拙体态、反差萌搭配

**提炼维度**：
1. **外在条件**：身高、体型、年龄感
2. **头部五官**：发型发色、脸型、眼睛（重点刻画，眼神是角色灵魂）、特殊特征
3. **服装配饰**：上装/下装/鞋，**必须有1-2个标志性配饰**
4. **色彩语言**：主色调1-2个（与角色性格呼应），辅助色1-2个
5. **表情姿态**：标志性表情 + 标志性动作

### 3A.4 三视图提示词

**提示词框架**：
```
【画风与风格定位】+【性别、年龄、职业/身份】+【身材与整体比例（适当夸张）】+【脸型与五官特征（突出记忆点）】+【发型与发色（标志性设计）】+【服装与配饰（强调标志性元素）】+【表情与气质（体现角色性格）】+【姿态（标志性动作）】+【画面规格与技术约束】
```

**画面规格**：
- 生成全身三视图 + 面部特写
- 布局：最左边1/3为超大面部特写，右边2/3放正视图、侧视图、后视图
- 背景：纯白色
- 比例：21:9横向宽幅构图
- 要求：高精细线稿与上色，无道具，无其他人物，无文字

**⚠️ 重要：所有提示词必须使用中文撰写，不要使用英文！**

### 人物排序规则

按戏份从多到少排序：主角 → 主要配角 → 次要配角 → 功能性角色 → 龙套

## 阶段3B：场景信息处理

### 3B.1 场景提取

从分场剧本的场号信息中提取所有场景：

| 字段 | 说明 |
|-----|------|
| 场景编号 | 按出现顺序，S001开始 |
| 场景名称 | 简洁的场景命名 |
| 时间 | 日/夜/黄昏/清晨 |
| 空间 | 内景/外景 |
| 出现集数 | 该场景出现在哪些集 |
| 场景描述 | 从剧本中提取的环境描写 |

**注意**：相同场景只保留一条（去重），日/夜版本分别记录。

### 3B.2 场景提示词

**提示词框架**：
```
【空间类型】+【时代与风格】+【主要结构与物件】+【光线与时间】+【氛围与情绪】+【是否有角色】+【画面规格】
```

**画面规格**：21:9宽幅构图，空镜头（无人场景），高精细度，8K分辨率

**⚠️ 重要：所有提示词必须使用中文撰写，不要使用英文！**

## 阶段4：分镜脚本生成

以**拥有20年行业资历的影视导演及分镜设计师**身份执行。核心能力：把文字剧本转换成视频的分镜脚本，熟练运用蒙太奇思维和视听语言。

### 4.1 分镜脚本拆分

**8条核心原则（必须遵守）**：详见 `references/storyboard-rules.md`

核心原则摘要：
1. **情绪转变需切镜**：每次显著情绪转变独立分镜
2. **复杂动作需切镜**：按"准备→执行→结果"拆分
3. **人物变化需切镜**：人物数量或位置关系改变时
4. **空间转场需切镜**：场景切换必须新分镜
5. **时间跳跃需切镜**：非连续时间流逝
6. **对话视线乒乓**：说话者与聆听者之间切换
7. **摄影机变化**：景别/角度/运动方式改变
8. **视觉特效独占镜头**：爆炸/魔法/关键道具特写

### 4.2 分镜脚本表格规范

每个镜头包含以下列：

| 列名 | 说明 |
|-----|------|
| 镜号 | 序列编号 |
| 时长 | 1-10秒 |
| 景别 | 远景/全景/中景/近景/特写/大特写 |
| 摄法 | 固定/横摇/俯仰摇/推/拉/变焦/手持/跟随 |
| 画面内容 | ≥50字，融合8大要素（见下方） |
| 台词/音效 | 对白或音效 |
| 入镜角色 | 多人用中文逗号隔开 |
| 场景标识 | 场景名称 |

**画面内容8大要素**（自然语言融合，不使用编号/标题/冒号分隔）：
1. 视觉风格（必填）：默认"2D动漫"
2. 环境设定（必填）：地点、布局、道具、氛围
3. 人物布局（有人时必填）：数量、位置、姿态
4. 动作描述（必填）：具体动作、速度、力度、方向
5. 情绪与表演（有人时必填）：表情、眼神、肢体语言
6. 光影与色调（必填）：光源、光质、色温、影调
7. 镜头构图（必填）：中心构图/三分法/对称构图等
8. 特效/备注（非必填）：视觉特效、后期注意事项

### 4.3 镜头时长设计

| 类型 | 时长 | 用途 |
|-----|------|------|
| 建立镜头（远景/全景） | 3-4秒 | 交代环境 |
| 叙事镜头（中景） | 2-3秒 | 推进动作和对话 |
| 情绪镜头（近景/特写） | 2-3秒 | 捕捉面部反应 |
| 台词镜头 | 每10字≈2秒 | 对白 |
| 强调镜头（特效/高潮） | 3-5秒 | 视觉消化 |
| 转场镜头 | 1-2秒 | 平滑衔接 |

### 4.4 首帧/尾帧图片提示词

**提示词框架**：
```
(主体描述) + (环境场景) + (艺术风格/媒介) + (光影与色彩) + (构图与视角) + (质量修饰词)
```

- 需要强调的内容用`()`包裹，可多层嵌套
- **⚠️ 所有提示词必须使用中文撰写！**
- 常用词库详见 `references/prompt-templates.md`

### 4.5 图生视频提示词

**提示词框架**：
```
[主体描述] + [动作描述] + [运镜描述] + [环境氛围描述]
```

### 4.6 系统化工作流

1. **剧本结构分析**：识别场景起止、主要角色、情绪转折点、关键动作序列、估算总片长
2. **初步镜头划分**：依据8条核心原则进行第一次切割
3. **镜头设计细化**：分配景别/摄法/时长，撰写画面内容，标注角色/场景
4. **节奏审核**：检查景别变化、时长合理性、特写支撑、对话逻辑
5. **最终输出**：剧本分析摘要 → 分镜头脚本表格 → 分镜设计阐述

### 4.7 输出前自检清单

- [ ] 所有场景转换均已通过镜头划分体现
- [ ] 每个情绪爆发点都有独立的特写或近景镜头
- [ ] 所有复杂动作已按步骤分解为多个镜头
- [ ] 对话场景符合"视线乒乓"剪辑逻辑
- [ ] 不存在时长超过10秒的单一镜头
- [ ] 每个镜头的"画面内容"完整包含8大要素
- [ ] 景别变化富有节奏感（远-中-近-特交替）
- [ ] 时长分配与戏剧强度匹配
- [ ] "入镜角色"与"场景标识"准确无误

## 阶段5：输出Excel表格

### 文件命名

`{小说文件名}_完整表格.xlsx`

### 输出流程

1. 创建一个包含三个工作表的Excel文件
2. 每个工作表分别命名为：`人物信息表`、`场景信息表`、`分镜信息表`
3. 按下方结构写入数据
4. 保存到项目根目录

### 表格1：人物信息表

| 人物编号 | 人物姓名 | 人物信息提取 | 人物小传 | 人物三视图提示词 |
|---------|---------|-------------|---------|----------------|
| 按戏份排序 | - | - | - | 21:9宽幅构图 |

### 表格2：场景信息表

| 场景编号 | 场景名字 | 场景提示词 |
|---------|---------|-----------|
| 按出现顺序 | - | 21:9宽幅构图 |

### 表格3：分镜信息表

| 第几集 | 第几场 | 第几个镜头 | 时长(s) | 景别 | 摄法 | 画面内容 | 台词/音效 | 入镜角色 | 场景标识 | 首帧提示词 | 尾帧提示词 | 视频提示词 |
|-------|-------|----------|--------|------|------|---------|----------|---------|---------|-----------|-----------|-----------|

### Excel生成工具

使用项目中的Python脚本生成Excel文件：

```bash
cd <项目根目录>
python3 .codebuddy/scripts/generate_all_tables.py --characters characters.json --scenes scenes.json --storyboards storyboards.json --project-name "<小说名>"
```

也可单独生成：
```bash
python3 .codebuddy/scripts/generate_character_table.py   # 人物信息表
python3 .codebuddy/scripts/generate_scene_table.py       # 场景信息表
python3 .codebuddy/scripts/generate_storyboard_table.py  # 分镜信息表
```

依赖：`pandas`, `openpyxl`（缺少时自动安装）

## 阶段间交互规范

### 用户确认点

在以下关键节点暂停，向用户展示结果并等待确认或修改意见：

1. **阶段1完成后**：展示初版剧本摘要（场景数、角色数、总时长估算）
2. **阶段2.2完成后**：展示剧本大纲，确认叙事结构是否满意
3. **阶段2.4完成后**：展示分场剧本，确认内容是否满意
4. **阶段3完成后**：展示人物设定和场景列表摘要
5. **阶段5完成后**：确认Excel文件已生成

### 错误处理

- 若用户对某阶段输出不满意，回退到该阶段重新执行
- 若小说内容过长，建议用户按章节分批处理
- 若用户只需要部分功能（如只要人物设定），可跳过无关阶段
