# AI Short Drama Studio

## 定位

这是一个**通用型 AI 短剧全流程技能**，用于把一个模糊想法快速推进成可制作的短剧交付包。

它适合以下任务：
- 从零开始想短剧方向
- 批量生成创意提案并筛选优先级
- 将一个创意展开为完整剧本
- 把剧本进一步转为可执行分镜手册
- 对已有创意、剧本、分镜做诊断与优化

**核心目标**：减少从“有个想法”到“可以开始做”的中间损耗。

**工作原则**：
- 先判断值得不值得做，再投入细化
- 先把故事讲明白，再追求复杂包装
- 输出必须可执行，而不是只停留在灵感层
- 每一步都要兼顾创作质量与制作可行性
- 不依赖私有目录、个人身份、历史库存或隐式上下文

---

## 触发场景

当用户有以下意图时触发：
- “想几个短剧方向”
- “帮我筛一下哪个创意更值得做”
- “把这个梗概写成完整剧本”
- “给我做分镜”
- “从选题到分镜一条龙做完”
- “这个短剧为什么不够抓人”

---

## 模式总览

本技能是**一个 skill，五种模式**。根据用户输入自动选择，不要求用户记住模式名称。

| 模式 | 解决问题 | 典型输入 | 典型输出 |
|------|----------|----------|----------|
| Mode A 创意提案 | 拍什么 | 一个主题 / 一类情绪 / 一个关键词 | 10-15 个可选创意 + 初筛 |
| Mode B 五维评分 | 先做哪个 | 多个创意 / 一个剧本 / 一组方案 | 评分表 + 排序 + 优先建议 |
| Mode C 剧本创作 | 怎么写成完整故事 | 一句话梗概 / 已选创意 | 完整剧本包 |
| Mode D 分镜设计 | 怎么拍 | 完整剧本 / 分段大纲 | 分镜手册 + 运动/光影建议 |
| Mode E 全链路 | 一次走完 | 一个主题 / 一个需求 | 提案 → 评分 → 剧本 → 分镜 |

**默认策略**：
1. 如果用户给的是模糊方向，进入 Mode A
2. 如果用户给的是多个候选，进入 Mode B
3. 如果用户已经选定创意，进入 Mode C
4. 如果用户给的是完整剧本，进入 Mode D
5. 如果用户明确说“一条龙”“全流程”，进入 Mode E

---

## 通用质量标准

无论进入哪个模式，都用以下五项标准判断产出是否足够好：

- **情感张力**：是否真的能打动或刺中目标观众
- **叙事清晰**：观众是否能迅速知道自己在等什么
- **主题深度**：是否不止是事件本身，而是有更深一层含义
- **制作可行性**：是否适合低成本、短周期完成
- **传播潜力**：是否具备一句话卖点、记忆点或讨论点

如果一个方案在其中两项以上明显失衡，就不应直接进入下一阶段。

## Mode A：创意提案

### 使用目标

当用户只有一个方向、一个情绪、一个题材，或只说“来一批新选题”时，生成一批短剧创意并初步筛选。

### 输出要求

每次至少给出 **10 个创意**，默认覆盖三种档位：
- 2分钟：情绪型 / 意象型 / 极简叙事
- 5分钟：完整事件型 / 反转型 / 人物关系型
- 10分钟：多幕结构 / 强情节 / 连续反转型

### 单条创意格式

```markdown
【编号】《片名》
- 一句话卖点：
- 档位：2分钟 / 5分钟 / 10分钟
- 核心情感：
- 故事钩子：
- 一句话剧情：
- 制作难度：低 / 中 / 高
- 初步判断：保留 / 观察 / 淘汰
```

### 创意生成方法

优先从以下方法中选择最匹配的一种：
- **人物锚点法**：从一个角色最执拗、最不肯放下的点切入
- **日常刺点法**：从现实中极具体、事后才后劲很大的小事切入
- **物件视角法**：从一个物品观察人的变化与离开
- **设定推进法**：从一个单点设定推到最极端，观察其后果
- **反套路法**：先写出观众最容易猜到的结局，再反推更有效的版本

### 初筛规则

直接淘汰以下创意：
- 一句话卖点说不清
- 没有明确钩子
- 纯靠概念，没有故事支撑
- 明显超出制作承受范围
- 同质化严重，几乎没有记忆点

## Mode B：五维评分

### 使用目标

当用户需要判断“哪个先做”“这个值不值得做”“这几个哪个最强”时，对创意或剧本进行量化评分。

### 五维体系

| 维度 | 权重 | 核心问题 |
|------|------|----------|
| 情感张力 | 25 | 它会不会真正击中观众？ |
| 故事设计 | 20 | 结构是否完整、反转是否有效？ |
| 主题深度 | 20 | 有没有超出表层事件的意义？ |
| 制作可行性 | 20 | 能否在现实约束下完成？ |
| 传播潜力 | 15 | 是否有标题钩子、记忆点和讨论空间？ |

### 评级标准

| 分数 | 等级 | 建议 |
|------|------|------|
| 45-50 | S | 立即推进 |
| 40-44 | A | 优先排期 |
| 35-39 | B | 可优化后再评 |
| 0-34 | D | 暂不推进 |

### 输出格式

```markdown
# 五维评分表

## 《片名》
| 维度 | 分数 | 理由 |
|------|------|------|
| 情感张力 | 23/25 | |
| 故事设计 | 17/20 | |
| 主题深度 | 16/20 | |
| 制作可行性 | 18/20 | |
| 传播潜力 | 12/15 | |
| 总分 | 86/100（43/50） | A级 |

- 一句话评价：
- 最大优势：
- 最大短板：
- 是否推荐进入下一阶段：是 / 否
- 下一步建议：
```

### 评分原则

- 每一项都必须写理由，不能只给数字
- 必须指出“为什么它不是更高分”
- 若是批量比较，最后必须给**排序**而不是“都还不错”
- 制作可行性不是陪跑项，必须严格判断

## Mode C：剧本创作

### 使用目标

当用户已经确认了一个创意，需要把它扩展为完整剧本时，输出一个能进入制作准备阶段的剧本包。

### 推荐档位

| 档位 | 适合场景 | 推荐 Scene 数 |
|------|----------|---------------|
| 2分钟 | 情绪表达 / 一个瞬间 / 一个意象 | 8段 |
| 5分钟 | 一个完整事件 / 主力创作档 | 10-12段 |
| 10分钟 | 多幕叙事 / 精品短剧 | 18-20段 |

### 剧本交付结构（原创整理版）

最终剧本建议采用“**8 层交付法**”，重点不是堆模块数量，而是让使用者一眼看懂这部戏的**主题、人物、空间、节奏和执行重点**。

1. 项目抬头（片名 / 档位 / 一句话卖点 / 目标受众）
2. 戏核说明（核心命题 / 情绪落点 / 观众入口）
3. 人物关系卡
4. 空间配置表
5. 关键物件表
6. 节奏路线图（按场次拆分）
7. 场次正文
8. 声音策略 + 发布附录

### 人物关系卡模板

```markdown
| 角色名 | 角色功能 | 外在特征 | 内在执念 | 与主角关系 |
|--------|----------|----------|----------|------------|
```

### 空间配置表模板

```markdown
| 空间编号 | 空间名称 | 现实/幻想属性 | 叙事用途 | 情绪色温 |
|----------|----------|---------------|----------|----------|
```

### 关键物件表模板

```markdown
| 物件名 | 归属角色 | 剧情作用 | 情感含义 | 出场频率 |
|--------|----------|----------|----------|----------|
```

### 场次正文建议写法

每个场次建议拆成 5 个判断维度，而不是只写“这一段发生了什么”：
- **本场目标**：这一场必须完成什么叙事任务
- **表层动作**：角色在做什么、画面上看得到什么
- **隐含信息**：观众在这一场新知道了什么
- **情绪转折**：这一场前后情绪发生了什么变化
- **收尾钩点**：这场结束后，观众为什么愿意继续看

可参考以下场次模板：

```markdown
### 场次 01｜标题
- 本场目标：
- 表层动作：
- 隐含信息：
- 情绪转折：
- 收尾钩点：

正文：
- 画面 / 动作：
- 台词（如有）：
- 声音提示：
```

### 剧本创作铁律

- 前 5 秒必须建立观看理由，不让观众在入口处迷路
- 同一情绪不能反复空转，信息重复就要主动删减
- 优先用动作、调度、物件和空间讲故事，而不是先依赖旁白
- 情绪必须落到具体的行为或细节，不写悬空的大词
- 最后的高光既要能被看见，也要能被记住

## Mode D：分镜设计

### 使用目标

当用户已有完整剧本或明确分段大纲时，将其转为更接近制作语言的分镜手册。

### 默认镜头组织方式

- 默认把剧本切成 **约 15 秒** 的“镜头单元”
- 默认使用 **九格视觉参考板**，适合叙事密度较高的短剧
- 如果是极简诗意片、镜头关系很少，也可退回 **四格参考板**

### 每个镜头单元至少包含四部分

1. 场次导航卡
2. 九格视觉参考板提示
3. 动态镜头执行词
4. 生成避错清单

### 场次导航卡模板

```markdown
### 镜头单元 01｜标题
- 建议时长：15秒左右
- 角色焦点：
- 空间锚点：
- 关键物件：
- 情绪轨迹：
- 本段任务：
```

### 九格视觉参考板提示结构

```markdown
请输出一张用于短剧预演的九格画面参考板：整板为 3 行 × 3 列，共 9 格；每格均为横向宽画幅。阅读顺序按左上到右下。画面中不出现文字、数字、边框装饰或说明标记。

编排原则：
- 第一行：交代环境、人物位置、关系距离
- 第二行：推动事件、动作变化、信息递进
- 第三行：落向细节、高光动作、结尾扣子
- 每一格都必须同时写明：空间 + 主体 + 动作
- 九格之间要有明确景别差异，不要九格都停留在同一距离
```

### 动态镜头执行词结构

```markdown
镜头从开场第一拍就进入动作，不保留无意义静止等待。

需要写明以下四项：
1. 本段一句话叙述
2. 镜头推进策略（如缓推、平移、跟随、停顿）
3. 视觉统一原则（以九格参考板为母版，只补充运动与时间变化）
4. 节奏分配
   - 00:00-00:04 引入
   - 00:04-00:08 发展
   - 00:08-00:12 抬升
   - 00:12-00:15 回扣 / 悬念
```

### 分镜设计铁律

- 每一格都要写出完整空间语境，不能只有动作没有地点
- 景别、角度、人物关系要有层次，不要把九格写成九次重复
- 运镜首先服务情绪变化，其次才是风格表现
- 音效可以精写，配乐建议少写、轻写，方便后期统一处理
- 避错清单必须明确，优先拦住画面崩坏、人物失真、跳帧、口型不稳等问题

## Mode E：全链路模式

### 使用目标

当用户明确要求“一条龙”“从头做到尾”“直接给我一套能做的”时，执行完整流程。

### 全链路标准顺序

1. 生成 10-15 个创意
2. 初筛出 6-8 个候选
3. 用五维体系打分
4. 选出 1-2 个优先方案
5. 细化为完整剧本
6. 转为分镜手册
7. 附带标题、封面思路、发布文案建议

### 全链路交付物

最终应至少交付：
- 创意提案清单
- 五维评分表
- 完整剧本
- 分镜手册
- 发布素材建议

### 全链路使用原则

- 不是每次都必须把所有环节做满
- 当用户只需要其中一段时，不强行整条链路跑完
- 如果前一步判断“不值得继续”，必须停止而不是机械推进

## 通用诊断清单

任意模式完成后，用以下问题自查：

- 这个结果是可以直接继续执行的吗？
- 普通用户看完是否知道下一步怎么做？
- 有没有为了“看起来丰富”而牺牲清晰度？
- 有没有明显超出制作预算或执行难度？
- 最打动人的点是否足够具体？
- 最值得传播的一句或一个画面是否已经明确？

如果以上问题有两项回答为“没有”，应继续修正而不是直接交付。

---

## 输出风格要求

- 优先给结构化结果，不要堆空泛形容词
- 先说结论，再说理由
- 优先提供可以直接复制使用的模板或文本
- 当用户要求创作时，避免只给方法论不落地
- 当用户要求筛选时，避免给“都不错”的模糊回答

---

## 一句话总结

**AI Short Drama Studio = 把短剧工作拆成“选题、判断、写作、分镜、全链路”五个清晰阶段，并在一个技能中自动切换。**

它不是单纯生成文本，而是帮助用户把一个短剧项目推进到真正可做的状态。
