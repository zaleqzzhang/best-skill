# Earnings One-Pager · 一页纸读懂财报

## Overview

将一家上市公司的财报信息浓缩为一张精美的暗色科技风 HTML 海报 + 一份结构化 Markdown 文档，融合四大信息源（财报原文、电话会议、卖方研报、自媒体解读），让用户 3 分钟快速掌握一份财报的核心要点，辅助投资决策。

支持全球主要市场：A股、港股、美股及其他主要交易所。

## 触发条件

当用户的请求符合以下任一情形时加载此 skill：
- 明确提到"财报"、"业绩"、"季报"、"年报"、"中报"、"earnings"等关键词，并要求分析或解读
- 使用"一页纸"、"海报"、"一图读懂"等表述
- 提供了财报 PDF 文件并要求总结
- 要求对某公司最新财务数据做可视化概览
- 输入公司名/股票代码后要求做财报分析

## 工作流

### Phase 1：确认目标

1. 解析用户输入，提取公司名称、股票代码
2. 自动识别所属市场：
   - A股：6 位数字代码（600xxx=沪市, 000xxx/002xxx=深市, 300xxx=创业板, 688xxx=科创板）
   - 港股：5 位数字代码（如 00700.HK）
   - 美股：字母代码（如 AAPL, TSLA）
3. 确认财报周期——若用户未指定，通过搜索确认最近一期已发布的财报
4. 若用户提供了 PDF 文件，先读取文件提取结构化数据；否则通过 web 搜索获取

> **智能识别**：用户可以直接说"解读苹果财报"、"腾讯最新财报"、"比亚迪2025年报"等自然语言，自动识别目标公司和时间。

### Phase 2：信息采集（四源并行）

按以下顺序搜索，每个来源独立执行，后续汇总。参考 `references/workflow-guide.md` 获取搜索关键词模板和各市场的特殊搜索策略。

**Source 1 — 财报核心数据**
- 搜索公司官方财报/业绩公告
- 提取：营收、净利润、EPS、毛利率、经营利润率、自由现金流、分业务收入、同比变化
- 与市场一致预期（consensus）对比：Beat / Meet / Miss
- A股特别关注：扣非净利润、经营现金流净额
- 港股特别关注：中期股息、全年股息
- 美股特别关注：Non-GAAP EPS、GAAP vs Non-GAAP 差异

**Source 2 — 电话会议纪要**
- 搜索 earnings call transcript / 业绩说明会纪要 / 业绩发布会
- 提取：管理层对业绩的解释、前瞻指引（guidance）、关键 Q&A、措辞变化
- 挑选 2-3 条最有价值的管理层原话（中英文均可，展示时用中文）
- A股/港股：搜索业绩说明会、分析师交流会纪要

**Source 3 — 卖方研报**
- 搜索主流投行/券商在财报后的研报摘要
- 提取：评级变化、目标价调整、核心观点
- A股：关注中金、中信、国君、华泰、招商等头部券商
- 港股：关注中金、摩根士丹利、高盛、瑞银等
- 美股：关注大摩、高盛、摩根大通、美银、花旗等
- 整理为简洁的观点列表 + 评级汇总表

**Source 4 — 媒体与自媒体解读**
- 搜索高质量财经媒体和自媒体解读文章
- A股渠道：微信公众号、雪球、东方财富、同花顺
- 港股渠道：智通财经、格隆汇、雪球
- 美股渠道：Seeking Alpha、Bloomberg、华尔街见闻、36氪
- 提取：独特的分析视角、市场情绪、与官方叙事不同的观点

> **重要**：不得捏造任何数据或信息。搜索不到的内容如实标注"未获取到"。每条信息需标注来源。

### Phase 3：分析研判

基于采集到的四源信息，进行以下分析：

1. **Verdict 判定**：参考 `references/workflow-guide.md` 中的 Verdict 判定规则，综合营收/EPS/指引表现给出 Beat / Meet / Miss 判定
2. **核心亮点提炼**：从四源中提取 4-6 条最重要的积极信号
3. **风险与隐忧**：提取 3-5 条需要关注的风险点
4. **分业务拆解**：按业务线/地区拆解收入构成和增长情况
5. **前瞻展望**：综合管理层指引 + 分析师预期，给出下季度/全年展望
6. **信号标签**：提炼 6-10 个关键信号标签，分为 positive / negative / neutral
7. **一句话总结**：用一句话概括这份财报最核心的信息

### Phase 4：生成双格式输出

#### 4A — 生成 HTML 海报

1. 读取 HTML 模板：`assets/earnings-poster-template.html`
2. 用 Phase 3 的分析结果填充所有 `{{PLACEHOLDER}}` 占位符
3. 遵循以下模板填充规则：

| 占位符 | 内容 | 说明 |
|--------|------|------|
| `{{BADGE_TEXT}}` | 如 `EARNINGS · FY2025 Q4` | 大写英文 |
| `{{COMPANY_NAME}}` | 如 `特斯拉 TSLA` | 中文名 + 代码 |
| `{{PERIOD}}` | 如 `2025年第四季度财报` | 中文周期 |
| `{{SUBTITLE}}` | 一句话定性 | 如"营收利润双双超预期，自动驾驶业务成新增长极" |

| `{{VERDICT_SUMMARY}}` | 2-3 句话概括表现 | |

### Phase 5：输出交付

1. 将生成的 HTML 文件和 Markdown 文件保存到用户工作目录
2. 使用 `preview_url` 预览 HTML 海报
3. 简要向用户总结：
   - 财报整体表现（Beat/Meet/Miss）
   - 最值得关注的 2-3 个要点
   - 信息采集覆盖度（哪些源获取成功，哪些缺失）
   - 已生成的文件列表

## 输出质量标准

### 信息密度
- 一张海报 = 一份完整的财报速览，信息量等价于一篇 2000 字的深度分析
- 每个区块的内容都要有实质信息，不要泛泛而谈

### 数据准确性
- 所有财务数字必须来自可靠信源，不可推测或编造
- 同比/环比变化必须注明基数
- 超/不及预期的百分比必须基于实际 consensus 数据
- 不同市场使用对应的财务准则：A股=中国会计准则，美股=GAAP/Non-GAAP，港股=IFRS

### 分析深度
- 不止是罗列数字，要有"so what"——数字背后意味着什么
- 亮点和风险要有具体论据支撑
- 管理层原话选取要有代表性，反映战略方向

### 视觉呈现
- HTML 必须是自包含单文件，直接浏览器可打开
- 暗色科技风格，排版紧凑但不拥挤
- 适当使用颜色编码：绿色=积极，红色=消极，琥珀色=中性/关注

### 本地化适配
- A股：使用中文财务术语（营业收入、归母净利润、扣非净利润等）
- 港股：使用港股常用术语（中期业绩、末期股息等），金额单位为港币
- 美股：使用标准英文缩写（Revenue, EPS, FCF 等），金额单位为美元
- 所有解读文字统一使用中文
